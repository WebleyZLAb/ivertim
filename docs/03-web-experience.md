# 03 — Web Experience

## Informaciona arhitektura

- Home
- Usluge
- Projekti
- Proces saradnje
- Blog / Savjeti
- Kontakt

Svaka usluga i svaki projekat imaju zaseban URL.

## Home

Redoslijed logike:

1. jasnoća — ko je Ivertim i šta radi;
2. diferencijacija — zašto je proces drugačiji;
3. dokaz — projekti, proces, proizvodnja i rezultati;
4. proces — kako saradnja izgleda;
5. izlazi — usluge, projekti, proces, blog i kontakt.

## Usluge

Svaka stranica ima jedno centralno pitanje i odgovara na:

- kome je usluga namijenjena;
- koji problem rješava;
- kako proces izgleda;
- šta korisnik treba pripremiti;
- koji su tipični rizici;
- koji je sljedeći korak.

## Projekti

Svaki projekat prikazuje:

- početni problem;
- ograničenja prostora;
- način razmišljanja;
- rješenje;
- rezultat;
- povezanu uslugu ili proces.

## Proces saradnje

Proces se prikazuje kao jasan timeline, bez mistifikacije.

Važno je objasniti:

- zašto se mjeri;
- kada se donose odluke;
- šta korisnik treba pripremiti;
- šta Ivertim ne pretpostavlja;
- kada se nešto potvrđuje.

## Blog / Savjeti

Sadržajna raspodjela:

- 50% edukativno;
- 30% prodajno / odluke;
- 20% ostalo.

Blog je važan za SEO, IEO i izgradnju povjerenja.

## Kontakt

Kontakt je mini landing stranica.

Primarni ishod:

- poslati upit;
- ili zaključiti da Ivertim nije pravi izbor.

## CTA

Jedan primarni CTA u navigaciji. Radni naziv: **Razgovarajmo**.

## UX principi

- svaka stranica ima jedno centralno pitanje;
- svaka sekcija je mikro-odgovor;
- otvoreni loop se koristi osim na Kontakt stranici;
- jasnoća ima prednost nad kreativnošću;
- mobilni prikaz nije umanjeni desktop, nego prilagođen tok odluke.
